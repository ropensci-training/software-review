{eph}: Use an automatic code formatter (review - commit).

Please use an automatic code formatter (e.g., usethis::use_tidy_style()). It‚Äôs useful to see that the style is consistent and follows a popular style guide like the ‚Äútidyverse style guide‚Äù (https://style.tidyverse.org/). This helps avoid thinking about style and saves mental energy for noticing more important things. Here are some examples of style violations:

{eph}: Show the output of examples in README (review - commit).

In the ‚ÄúUsage‚Äù section, please show the results so they can be seen without needing to install the package and run the code.

{agroclimatico}: Rename functions (review - commit).

a slight name change could be considered to avoid confusion

{agroclimatico}: Group functions (review - commit).

A suggestion to make the overall functionality of the package clearer is to group the functions in the index by topic.

{karel}: Express the package‚Äôs need in README (review - commit).

Add statement of need. The Who is Karel? section of the README hints at the need but does not describe it explicitly.
